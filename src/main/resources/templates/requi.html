Requisitos del Sistema

Requisitos Funcionales

  • HUA.21. Calificar arrendatario
  • HUA.20. Calificar finca
  • HUA.19. Pagar arriendo
  • HUA.18. Ver solicitudes de arriendo
  • HUA.17. Solicitar arriendo
  • HUA.16. Buscar propiedades
  • HUA.15. Autenticación
  • HUA.14. Activación de cuenta
  • HUA.13. Creación de cuenta
  • HUA.12. Calificar arrendatario

Requisitos No Funcionales

  • Rendimiento: El sistema debe ser capaz de manejar hasta 10,000 usuarios concurrentes.
  • Seguridad: Todas las transacciones deben estar encriptadas usando TLS.
  • Usabilidad: La interfaz de usuario debe ser intuitiva y fácil de usar.
  • Escalabilidad: El sistema debe ser escalable para acomodar el crecimiento futuro.
  • Mantenibilidad: El código debe seguir las mejores prácticas para facilitar el mantenimiento.
  • Disponibilidad: El sistema debe tener una disponibilidad del 99.9%.
